### 2026年05月18日

- **重构**：将 ~1970 行 `hostconfig.py` 拆分为 `hcm/` 子目录 9 个功能模块
  - `models.py` — 纯数据类（ConfigSnapshot / UpdateRecord / LibsConfig）
  - `imports.py` — 集中导入 func 子模块（无 jpfuncs）
  - `collector.py` — 配置收集器
  - `storage.py` — JSON 持久化 + 合并 + 智能保存
  - `markdown.py` — Markdown 解析/生成纯函数
  - `joplin_sync.py` — Joplin API 薄封装 + 笔记更新
  - `sync_service.py` — 编排器
  - `utils.py` — 辅助函数 + jpfuncs 导入守卫
- **去除 mock 机制**：func 子模块不存在直接报错，不做降级
- **jpfuncs 导入守卫**：`utils.py` 的 `_ensure_jpfuncs()` 首次导入失败后注入 mock 模块到 `sys.modules`，确保 `jpapi = getapi()` 副作用只执行一次
- **文档**：更新 `docs/技术手册.md` 反映新架构
